# Kanban Board Explanation

## What is a Kanban Board?

A Kanban board is a visual project management tool used to track tasks as they move through different stages of a workflow. Tasks are represented as cards, and columns represent the stages of progress.

---

## How the Kanban Board Works in This Project

The Smart Campus System uses a customized Kanban board with the following columns:

- To Do – Tasks that have not yet started  
- In Progress – Tasks currently being worked on  
- Testing – Tasks being tested for quality assurance  
- Blocked – Tasks that cannot proceed due to dependencies or issues  
- Done – Completed tasks  

---

## Workflow Visualization

Each column represents a stage in the development process. Tasks move from left to right, allowing the team to easily track progress and identify bottlenecks.

---

## Work-In-Progress (WIP) Limits

To improve productivity and avoid overload:

- A maximum of **3 tasks** is allowed in the "In Progress" column.
- This ensures that developers focus on completing tasks before starting new ones.

---

## Support for Agile Principles

The Kanban board supports Agile development in the following ways:

- **Continuous delivery:** Tasks are completed incrementally.
- **Flexibility:** Tasks can be added or reprioritized at any time.
- **Transparency:** All team members can see the status of tasks.
- **Collaboration:** Tasks are assigned using GitHub Issues and @mentions.

---

## Customization Choices

Two additional columns were added:

- **Testing:** Ensures tasks are verified before completion.
- **Blocked:** Highlights tasks that cannot proceed due to dependencies.

These improvements ensure better workflow management and align the system with real-world development practices.

---

## Conclusion

The Kanban board provides a clear and efficient way to manage tasks, track progress, and support Agile development in the Smart Campus System project.
