# Template Analysis and Selection

## Introduction
GitHub Projects provides several templates that support software development workflows. These templates help teams organize tasks, track progress, and apply Agile methodologies effectively.

---

## Comparison of GitHub Project Templates

| Template | Columns / Workflow | Automation Features | Agile Suitability |
|----------|------------------|-------------------|------------------|
| Basic Kanban | To Do, In Progress, Done | No automation | Suitable for small projects but lacks advanced Agile support |
| Automated Kanban | To Do, In Progress, Done | Automatically moves issues when status changes | Highly suitable for Agile workflows and continuous delivery |
| Bug Triage | New, Needs Review, Closed | Auto-labeling and categorization of bugs | Best for bug tracking, not full development lifecycle |
| Team Planning | Backlog, Ready, In Progress, Done | Supports prioritization and scheduling | Suitable for sprint planning and structured Agile teams |

---

## Selected Template: Automated Kanban

The **Automated Kanban template** was selected for the Smart Campus Attendance and Appointment System.

---

## Justification

The Automated Kanban template aligns well with the project requirements because:

- It supports **continuous workflow**, which is a key Agile principle.
- It includes **automation features**, reducing manual work by automatically moving tasks across columns.
- It integrates seamlessly with **GitHub Issues**, which represent user stories from Assignment 6.
- It provides **clear visibility of task progress**, making it easier to track development stages.

Compared to Basic Kanban, it improves efficiency through automation. Compared to Bug Triage, it supports full system development rather than just issue tracking.

---

## Conclusion

The Automated Kanban template is the most suitable choice because it enhances productivity, supports Agile practices, and provides a structured workflow for managing the Smart Campus System project.
